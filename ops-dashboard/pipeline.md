## First Distribution Plan

Lead wedge: `restricted-party-screen`
Bundle / upsell: `vendor-entity-brief`

Primary buyer workflow targets:
1. Procurement intake for a new supplier before approval.
2. Vendor onboarding before the counterparty is added to payment or ERP systems.
3. Payout approval for a contractor, marketplace seller, or other new payee.
4. Compliance or risk triage for a fast first-pass name check.
5. Agentic vendor review where a cheap yes/no gate prevents deeper work on a bad counterparty.

Where Payments MCP fits:
- default invocation path for agent-side usage
- best for cheap, immediate screen calls
- best when the buyer is already in a tool-driven workflow

Where direct operator workflows fit:
- email, Slack, ticketing, spreadsheet, and approval-note workflows
- manual procurement or onboarding review
- copy-paste prompts when the buyer is not ready for tool integration

Opening message drafts to use:
1. `We built a very small counterparty screen for procurement and onboarding teams. It gives you a cheap first-pass gate before you spend time on a full review. If that would fit one of your approval flows, I can show you the 30-second version.`
2. `If your team screens new vendors, payees, or contractors before approval, this is meant to slot in as the first check. The bundle adds a short entity brief only when someone needs handoff context.`
3. `I'm reaching out because we have a low-friction screen for new counterparties that works well as a default gate in agent or operator workflows. If you want, I can send a copy-paste prompt or a live example.`

## Immediate Next Tests
