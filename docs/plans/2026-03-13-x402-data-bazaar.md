# x402 Data Bazaar Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a single Express server with 15+ paid API endpoints wrapping free public data sources, protected by x402 payment middleware, deployed to Vercel.

**Architecture:** One Express app, one deployment. Each data source is a route handler in `routes/`. Shared x402 middleware protects all routes. Upstream API keys stored as env vars. Deployed to Vercel via `vercel.json` serverless adapter.

**Tech Stack:** Express, x402-express, node-fetch, Vercel (deployment)

### Task 2: No-Key APIs (8 routes, fastest to build)

**Files:**
- Create: `routes/vin.js` (NHTSA VIN decoder)
- Create: `routes/weather.js` (Open-Meteo current + forecast)
- Create: `routes/holidays.js` (Nager.Date holidays by country + today check)
- Create: `routes/exchange-rates.js` (ExchangeRate-API)
- Create: `routes/ip.js` (ip-api.com geolocation)
- Create: `routes/food.js` (Open Food Facts barcode lookup)

Each route handler:
1. Parse query/path params
2. Fetch upstream API
3. Transform response to clean, agent-friendly JSON
4. Return with consistent `{ success, data, source }` envelope

**Upstream URLs:**
- VIN: `https://vpic.nhtsa.dot.gov/api/vehicles/DecodeVin/{vin}?format=json`
- Weather current: `https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current=temperature_2m,relative_humidity_2m,wind_speed_10m,precipitation,weather_code`
- Weather forecast: `https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weather_code&timezone=auto`
- Holidays: `https://date.nager.at/api/v3/PublicHolidays/{year}/{countryCode}`
- Exchange: `https://open.er-api.com/v6/latest/{base}`
- IP: `http://ip-api.com/json/{ip}`
- Food: `https://world.openfoodfacts.org/api/v2/product/{barcode}`
